## 第一部分：项目概述与优先级

### 1.1 三个 Deferred 项

| 优先级 | ID | 功能 | 复杂度 | 预计任务数 | 说明 |
|--------|-----|------|--------|-----------|------|
| P0 | D7 | Dashboard 远程访问（认证 + HTTPS） | 中 | 8 | 复用已有 Auth 基础设施，安全加固 Dashboard |
| P1 | D6 | Agent 协作模式（N-Agent 协同） | 高 | 10 | 从 DualAgent 扩展到 N-Agent 协作框架 |
| P2 | D3 | Tauri Desktop 桌面应用 | 低 | 6 | 包装现有 Dashboard 为原生桌面应用 |

### 1.2 优先级决策理由

1. **D7 (Dashboard 远程访问)** 排最高：Dashboard 已完成 (D2)，但仅绑定 localhost，无认证。远程访问是生产环境基本需求，且 `octo-engine` 已有完整的 ApiKeyStorage + RBAC + JWT 基础设施可直接复用。工作量最小，价值最大。
2. **D6 (Agent 协作)** 排第二：DualAgentManager (D5) 已证明双 Agent 模式可行，但硬编码为 Plan/Build 对。扩展到 N-Agent 需要新的 CollaborationManager + 消息传递层 + 共享上下文，是架构性升级。
3. **D3 (Tauri Desktop)** 排最后：Dashboard + 远程访问完成后，Tauri 只需 ~500 行 Rust 包装层。依赖 D7 的 HTTPS + Auth 完成后才有意义（否则桌面应用也只能访问 localhost）。

### 1.3 依赖关系

```
D7 (Dashboard Auth + HTTPS)
 ├─► D3 (Tauri Desktop) — D3 可复用 D7 的 TLS 配置
 └─► D6 (Agent Collaboration) — D6 的 Dashboard 面板需要 Auth 保护

D6 独立于 D3/D7 的 engine 层实现，但 Dashboard 集成部分依赖 D7
```

use axum_server::tls_rustls::RustlsConfig;

pub async fn run_dashboard(opts: DashboardOptions) -> Result<()> {
    let router = build_router(/* ... */);
    let addr = format!("{}:{}", opts.host, opts.port);

    if opts.tls_enabled {
        let tls_config = RustlsConfig::from_pem_file(
            &opts.cert_path.unwrap(),
            &opts.key_path.unwrap(),
        ).await?;
        axum_server::bind_rustls(addr.parse()?, tls_config)
            .serve(router.into_make_service())
            .await?;
    } else {
        let listener = TcpListener::bind(&addr).await?;
        axum::serve(listener, router).await?;
    }
    Ok(())
}

#### 2.4.2 Auth 中间件

```rust
// dashboard_auth.rs — 新文件
use axum::{extract::Request, middleware::Next, response::Response};
use octo_engine::auth::{ApiKeyStorage, UserContext, Role};

pub async fn auth_middleware(
    State(key_store): State<Arc<ApiKeyStorage>>,
    req: Request,
    next: Next,
) -> Result<Response, StatusCode> {
    let auth_header = req.headers()
        .get("Authorization")
        .and_then(|v| v.to_str().ok());

    match auth_header {
        Some(key) if key.starts_with("Bearer ") => {
            let api_key = &key[7..];
            match key_store.verify(api_key).await? {
                Some(stored_key) if !stored_key.is_expired() => {
                    let ctx = UserContext::new(
                        Some(stored_key.user_id.clone()),
                        vec![],
                        Some(stored_key.role.clone()),
                    );
                    req.extensions_mut().insert(ctx);
                    Ok(next.run(req).await)
                }
                _ => Err(StatusCode::UNAUTHORIZED),
            }
        }
        _ => Err(StatusCode::UNAUTHORIZED),
    }
}
```

## 风险与缓解

| 风险 | 严重度 | 缓解措施 |
|------|--------|---------|
| TLS 自签名证书浏览器警告 | 低 | 提供证书信任脚本 + 文档 |
| Auth 旁路导致数据泄露 | 高 | 非 localhost 绑定强制要求 `--require-auth` |
| N-Agent 协作死锁 | 中 | 每条消息设超时 + TurnGate 互斥 |
| Tauri WebView 兼容性 | 低 | 最低要求 macOS 12 / Windows 11 / Ubuntu 20.04 |
| 协作上下文膨胀 | 中 | SharedState 设 key 数量上限 + TTL |
| build_router 未导出为 public API | 低 | D3-2 前将 dashboard 路由构建函数提升为 pub |
