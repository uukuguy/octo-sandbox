## S1. L2 MCP Orchestrator 增强

### S1.T1 — L2 MCP 注册表 skill-dependency 解析 API

**目标**：L4 能查询 "给定一组 skill dependencies，返回对应的 MCPServerConfig 列表"

**涉及文件**：
- `tools/eaasp-mcp-orchestrator/src/routes.rs` — 新增 endpoint
- `tools/eaasp-mcp-orchestrator/src/manager.rs` — 新增方法
- `tools/eaasp-mcp-orchestrator/mcp-servers.yaml` — 注册 mock-scada + eaasp-l2-memory

**实现要点**：
1. 新增 `POST /v1/mcp/resolve` endpoint
   - 输入：`{ "dependencies": ["mcp:mock-scada", "mcp:eaasp-l2-memory"] }`
   - 输出：`{ "servers": [{ "name": "mock-scada", "transport": "stdio", "command": "mock-scada", "args": [] }, ...] }`
2. 从 `mcp-servers.yaml` 配置匹配 dependency name → McpServerDef
3. 将 McpServerDef 转换为 proto 兼容的 MCPServerConfig JSON 格式
4. 对于 SSE/HTTP transport，返回 URL；对于 stdio，返回 command + args

**测试**：3-4 个 Rust 测试（resolve 正常/未注册/混合 transport）

**前置**：无

---

### S1.T2 — L2 Memory Engine stdio MCP wrapper

**目标**：Memory Engine 可作为 MCP server 被 L1 runtime 通过 stdio/SSE 连接

**涉及文件**：
- `tools/eaasp-l2-memory-engine/src/eaasp_l2_memory_engine/mcp_server.py` — 新建
- `tools/eaasp-l2-memory-engine/pyproject.toml` — 添加入口点

**实现要点**：
1. 新建 `mcp_server.py`：用 `fastmcp` 或 `mcp` SDK 包装现有 6 个 MCP tools
2. 每个 tool handler 调用现有 `MemoryStore` 方法（复用 `mcp_tools.py` 的逻辑）
3. 支持 stdio transport（默认）+ SSE transport（`--transport sse --port <port>`）
4. 在 `pyproject.toml` 添加 `eaasp-l2-memory = "eaasp_l2_memory_engine.mcp_server:run"` 入口点
5. 注册到 `mcp-servers.yaml`

**测试**：3-4 个 Python 测试（tool list / invoke memory_search / invoke memory_write_anchor）

**前置**：无（可与 S1.T1 并行）

## S2. L4 编排集成

### S2.T1 — L4 L1RuntimeClient 新增 connect_mcp()

**目标**：L4 能通过 gRPC 调用 L1 的 ConnectMCP

**涉及文件**：
- `tools/eaasp-l4-orchestration/src/eaasp_l4_orchestration/l1_client.py` — 新增方法
- `tools/eaasp-l4-orchestration/tests/` — 新增测试

**实现要点**：
1. `L1RuntimeClient.connect_mcp(session_id: str, servers: list[dict]) -> ConnectMCPResponse`
2. 构建 `ConnectMCPRequest` proto：session_id + servers（从 L2 resolve 结果转换）
3. 调用 gRPC stub `ConnectMCP()`
4. 返回 connected/failed 列表
5. 错误处理：连接失败 → 抛异常，部分失败 → 日志警告但不阻塞

**测试**：2-3 个 pytest 测试（全成功 / 部分失败 / gRPC 不可达）

**前置**：无

---

### S2.T2 — L4 session create 集成 MCP 编排

**目标**：L4 在 Initialize 成功后，自动查询 L2 → 调用 ConnectMCP

**涉及文件**：
- `tools/eaasp-l4-orchestration/src/eaasp_l4_orchestration/session_orchestrator.py`
- `tools/eaasp-l4-orchestration/src/eaasp_l4_orchestration/mcp_resolver.py` — 新建

**实现要点**：
1. 新建 `McpResolver` 类：
   - `resolve(dependencies: list[str]) -> list[McpServerConfig]`
   - 调用 L2 MCP Orchestrator `POST /v1/mcp/resolve`
   - 对于 containerized runtime（hermes），将 stdio transport 转换为 SSE（host.docker.internal 规则）
2. 在 `create_session()` 中，Initialize 成功后：
   ```python
   mcp_deps = [d for d in skill_deps if d.startswith("mcp:")]
   if mcp_deps:
       servers = await self.mcp_resolver.resolve(mcp_deps)
       result = await self.l1_client.connect_mcp(session_id, servers)
       # 记录 SESSION_MCP_CONNECTED 事件
   ```
3. 事件追踪：写入 `SESSION_MCP_CONNECTED` event 到 session_events 表

**测试**：3-4 个 pytest 测试（有 MCP 依赖 / 无 MCP 依赖 / L2 不可达 / 部分失败）

**前置**：S2.T1（L1RuntimeClient.connect_mcp）
