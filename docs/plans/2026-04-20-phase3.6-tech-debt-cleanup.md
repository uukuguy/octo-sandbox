## Task Breakdown

### T1 — D140: grid-engine HookContext::with_event retrofit

**Precondition met**: Python 侧自 Phase 2 S3.T5 合规，Rust 侧 3–5 LOC 热修即可闭环。

**Action**:
1. 定位 `HookContext` struct 和 `with_event` builder (`crates/grid-engine/src/` 下 grep `HookContext`)
2. 按 ADR-V2-006 §2 envelope schema 补齐字段：`event`, `skill_id`, `tool_name`（如缺）
3. 改调用点：`execute_command` / hook dispatch sites（根据 ledger D120/D140 笔记，3–5 处）
4. 改 `to_json` / `to_env_vars` 序列化，确保 Rust stdin/env 与 Python 字节对齐
5. 添加/更新 `test_hook_envelope.py` Rust 侧断言（contract test 里面，跑 `make v2-phase3-contract-grid`）

**Sign-off**: `make v2-phase3-contract-grid` 绿，对比 `claude-code` 侧 envelope 无字段漂移。

**Est**: 30-60 min.

### T2 — D145: session_orchestrator delta_buf dedup

**Precondition met**: 纯 refactor，无下游契约依赖。

**Action**:
1. 定位 `send_message` + `stream_message` 两处 `delta_buf` / `ctype == "text_delta"` 判断
2. 抽 helper `_accumulate_text_delta(chunk_type, delta_buf, ...)`（具体签名待读代码决定）
3. 两处改调用
4. 跑 `pytest tools/eaasp-l4-orchestration/tests/ -xvs` 确保无回归

**Sign-off**: `grep -n "delta_buf" tools/eaasp-l4-orchestration/src/` 只剩 helper 一处定义 + 两处调用。

**Est**: 20-30 min.

### T3 — D147: Python proto .pyi stubs regenerate

**Precondition met**: Phase 3.5 已把 `common.proto` 的 `ChunkType` enum 落地。

**Action**:
1. 调研 `grpcio-tools` 生成带 IntFlag 的 stub 的正确 flag（可能需要 `--pyi_out` + 额外配置）
2. 重新生成 4 个包的 stub：`claude-code-runtime-python`, `nanobot-runtime-python`, `pydantic-ai-runtime-python`, `eaasp-l4-orchestration`
3. 验证 Pyright strict 不再报 `ChunkType | str | None` 拒绝 int

**Sign-off**: 4 个包编辑器打开时 Pyright 无相关报错。如果工具链限制无法彻底修，记 follow-up 到 D147 状态 + 降为 P3。

**Est**: 30-60 min（含工具链试错）。

### T4 — D150: build_proto.py 三合一

**Precondition met**: T3 完成后再做（避免重复工作）。

**Action**:
1. 读 3 份 `build_proto.py` 找出差异（ledger 说"仅包名不同"）
2. 写 `scripts/gen_runtime_proto.py --package-name <name> --out-dir <dir>`
3. 改 3 处 Makefile / README 调用
4. 删旧脚本
5. 跑一次 `scripts/gen_runtime_proto.py` 所有包，diff 对比生成内容与 T3 末尾一致

**Sign-off**: `grep -rn "build_proto.py" lang/ scripts/` 零匹配（除了历史 git log）。

**Est**: 30 min.

### T5 — D146: Pyright workspace config

**Precondition met**: 纯编辑器配置。

**Action**:
1. 定位现有 `pyrightconfig.json` 或 `.vscode/settings.json`
2. 为每个 Python 包（`lang/*-python/`, `tools/eaasp-*/`）添加 `extraPaths` 或 `venv` 指向 per-package `.venv`
3. 验证 VS Code / Cursor / Zed 里 `import` 全绿

**Sign-off**: 用户确认编辑器 import 不再报 unresolved。

**Est**: 15-30 min。
