### S0.T2 — ADR-V2-NEW: Agent loop 多步工作流设计原则

**问题**: D87 暴露 grid-engine agent loop 在 text-only 响应后终止。需要定义**标准化的 agentic loop 行为**，确保三 runtime 一致。

**2026-04-14 根因修正** — 见 `docs/design/EAASP/AGENT_LOOP_ROOT_CAUSE_ANALYSIS.md`：

> grid-engine harness.rs 的 loop 语义与 claude-code 等价（无 tool_use → terminate），**没有 bug**。claude-code MVP 能过是因为 **Claude Sonnet 4 自己主动连续调 tool，根本没触发 text-only 退出分支**。而 grid-runtime 用 OpenAI 模型，OpenAI 习惯调一次 tool 后问"要不要继续"，loop 真的终止。**hermes 在同样 OpenAI 模型上能多轮**，因为 hermes loop 有 `intermediate-ack detection + continuation 注入`，主动替用户回"继续"。

**修正后候选方向**:

| 选项 | 描述 | 推荐度 |
|------|------|------|
| **A. hermes intermediate-ack** | `stop_reason=EndTurn && tool_uses.is_empty() && total_tool_calls > 0 && workflow_continuation_count < 3` 时注入 `[System: Continue now. Execute the required tool calls...]` + `continue` | ⭐⭐⭐ 照抄已验证设计，skill 零修改 |
| **B. goose final_output tool gate** | 每个 skill 注册 `final_output` tool，LLM 必须显式调用才退出 | ⭐ 向后不兼容 |
| **C. skill frontmatter `min_tool_calls`** | 声明式下限 | ⭐ 需改 skill + schema |
| **D. 混合** | A 作为默认 loop 行为（向所有 skill 生效）+ B/C 作为 opt-in 强约束 | ⭐⭐ Phase 2 用 A，B/C 推迟 |

**ADR-V2-016 决定**: **A（hermes intermediate-ack 注入）** 作为 Phase 2 默认实现。B/C 推迟到 Phase 3+。

**前置**: 无，可与 S0.T1 并行

```rust
// harness.rs: L1168 退出分支入口前
if stop_reason == StopReason::EndTurn
    && tool_uses.is_empty()
    && total_tool_calls > 0                       // 已在 workflow 中途
    && workflow_continuation_count < MAX_WORKFLOW_CONTINUATIONS  // 默认 3，防死循环
{
    workflow_continuation_count += 1;
    messages.push(ChatMessage::assistant(&full_text));
    messages.push(ChatMessage {
        role: MessageRole::User,
        content: vec![ContentBlock::Text {
            text: "[System: Continue now. Execute the required tool calls \
                   to complete the workflow. Do not ask for confirmation.]".into(),
        }],
    });
    let _ = tx.send(AgentEvent::IterationEnd { round, ... }).await;
    continue;  // 不 break，重入 loop
}

// ↓ 保持原有 L1169 退出分支不变
if tool_uses.is_empty() { ... }
```

## S3. PreCompact + Skill Extraction

### S3.T1 — PreCompact hook 接入（含 hermes + claude-code 混合策略）⭐ 批次 B

**来源**: `docs/design/EAASP/AGENT_LOOP_PATTERNS_TO_ADOPT.md` #5 #6 #9

**涉及文件**:
- 拦截器层：context size 超阈值时触发 `PRE_COMPACT` 事件
- `crates/grid-engine/src/agent/context_compressor.rs` — 新建
- `crates/grid-engine/src/agent/harness.rs` — 集成 compactor

**策略组合**：

**Proactive（hermes tail-protected）**：
- 保护头部（system + 第一组 user/assistant）
- 保护尾部（最近 ~20K token）
- 中间用廉价模型（haiku / gpt-4o-mini）总结
- **下次压缩时复用上次 summary，不是从头总结**（关键节省）
- Summary 长度自适应：原内容 20%，2K-12K ceil
- Tool output pre-pruning 在 LLM 调用前

**Reactive（claude-code）**：
- 413 payload-too-large / context overflow 时才触发
- `hasAttemptedReactiveCompact` guard 防死循环
- Allow more aggressive truncation 在降级时

**跨压缩 token 预算**：
- `finalContextTokensFromLastResponse()`：从 `usage.iterations[-1]` 取真实 context 占用
- `taskBudgetRemaining` 是 loop-local 变量，**压缩后不重置**

**配置**：
