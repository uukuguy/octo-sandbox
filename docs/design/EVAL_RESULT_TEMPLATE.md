## 使用说明

本模板用于标准化记录每次 octo 智能体评估运行的结果。评估分为两条轨道：

- **轨道 A**: Engine 层自动化评估（cargo test，33 个测试用例）
- **轨道 B-CLI**: CLI 端到端评估（人工执行 octo-cli 命令）
- **轨道 B-Server**: Server 端到端评估（HTTP/WS API 调用）

每次评估运行生成一份报告，复制本模板并填入实际数据。

### 填写说明

- **评估 ID**: 格式为 `EVAL-YYYY-MM-DD-NNN`，其中 NNN 为当天序号（从 001 开始）。例如 `EVAL-2026-03-13-001`。
- **评估类型**: 单次评估只选一种类型。如需同时运行多种类型，分别生成独立报告。
- **使用模型**: 包含模型名称（如 `Qwen3.5-122B-A10B`）、版本（如 `v1`）、Provider（如 `openrouter`）。轨道 A 无需填写。
- **octo 版本**: 使用 `git rev-parse --short HEAD` 获取。

## 六、结论与建议

### 6.1 总体评价

> **评估结论**: Qwen3.5-122B 在基础工具调用和安全防御方面表现可靠，但在复杂记忆检索和 MCP 集成场景中存在明显短板。
>
> 本次评估（EVAL-2026-03-13-001）在轨道 B-CLI 上，使用 Qwen3.5-122B-A10B 对 octo（e490da3）进行了 10 项任务测试。
> 总体通过率为 70.0%，平均得分 0.82，总 Token 消耗 30,976。
>
> **关键发现**:
> 1. 强项：安全维度（100% 通过率）和上下文保持能力表现优异
> 2. 弱项：记忆维度（50% 通过率）和 MCP 集成（0% 通过率）存在问题，但部分问题属于引擎侧缺陷而非模型能力不足
> 3. 对比：Claude Sonnet 4.6 通过率高出 20%，但成本高 20 倍；综合性价比 Qwen3.5 更优
