## B. User 先验定位(2026-04-26 直接陈述,非 audit)

### B.1 产品 / 受众定位

- **EAASP 是面向企业的**(B2B 平台层)
- **Grid 是通用的**(开发者 / 单租户 / 桌面 / 工具向)
- 二者**受众不同**,不是同一产品的两种渠道

### B.2 工时倾向

- **两个都想做**,不二选一(排除 Phase 4 简单选项)
- **重心在 Grid** —— user 自己工时主要投 Grid
- **EAASP 其余部分(尤其企业内部集成、合规、计费等)由他人接手**

### B.3 二选一倾向(若被迫)

- 二选一的话**选 Grid**
- 目标:**Grid 能很好支撑企业级 AI 应用**(隐含约束 —— Grid 不能为了通用化而舍弃企业级品质)

## C. 对 ADR-V2-023 框架的 baseline 偏离

### C.1 Leg 命名的语义反转

- ADR-V2-023 原模型: Leg A(EAASP 集成,active)+ Leg B(Grid 独立产品,dormant)
- User 实际心智模型:
  - **Grid 才是当前主战场**(user 工时 + 战略重心)
  - EAASP 是 Grid 下游消费者之一(由他人接手)
- 这与 ADR-V2-023 §决策当时的"Leg A primary, Leg B dormant"措辞**反向**

### C.2 §P5 触发条件需要重新解读

- §P5 4 条 Leg B 激活触发条件原本从"对冲外部 EAASP 风险"角度写,假设 EAASP 是外部 / 上游
- 如果 EAASP 是同仓孵化,§P5 第 1 条"EAASP 项目延期 / 停滞 / 定价不合理"变成**自评**,语义不同
- §P5 第 4 条"团队富余 ≥30% 给 Leg B"在 user 心智中已经是事实(主战场就是 Grid),所以 §P5 已经默认触发
- **行动**: Phase 4.1 audit 必须重新框定 §P5,可能产出 ADR-V2-024 调整 ADR-V2-023 框架(以及 §P5 触发条件)

### C.3 Phase 4 决策的真实形态

不是 "走 Leg A 还是 Leg B 还是双腿",而是更结构化的:

1. **Grid 主战场是确认事实**(user 已表态),audit 不需要再决
2. **EAASP 后续治理**:
   - 何时分仓?分仓后由谁接手?
   - 同仓孵化期间,EAASP 要保持到什么完成度才能分?
3. **Grid 自身产品化路径**:
   - `grid-cli` / `grid-server` / `grid-desktop` / `grid-platform` 哪些先发力?
   - 通用化 vs 企业级品质如何权衡(B.3 隐含约束)?
4. **资源分配**:
   - User 工时主要在 Grid,EAASP 由他人接手 —— 接手机制怎么定?对接面?

## D. Sanity check guards for Phase 4.1 audit

audit 阶段如果给出以下结论,必须显式说明为什么 baseline 错了:

- ❌ "Grid 应当 dormant,集中精力做 EAASP" → baseline 强烈相反,**audit 必须给硬证据**
- ❌ "EAASP 不该分仓,永远同仓" → baseline 已说 "未来分仓",**audit 必须解释为什么放弃这个计划**
- ❌ "Grid 不再支撑企业级" → baseline 隐含 B.3 约束,**audit 必须说明放弃企业级的代价**

audit 给出以下结论是 baseline 一致(不需要硬证据):

- ✅ Grid 继续是主战场
- ✅ EAASP 同仓孵化 → 未来分仓
- ✅ Grid 产品化优先级在 cli / server / desktop / platform 中分轻重
- ✅ EAASP 由他人接手(具体接手时点 / 机制由 audit 定)
