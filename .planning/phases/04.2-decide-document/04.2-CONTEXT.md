- **Verdict**: **yes** (从 partial 收敛)
- **Answer**: **Grid 全栈 ≈60% / EAASP 引擎 ≈30% / 元工作 (planning/audit/governance) ≈10%**

### Open Item #3 — §F.Q4 grid-cli/server/desktop/platform/web* 五者发力顺序 (P1)

- **Verdict**: **yes** (Grid 优先 + 并行可行已锁, 五者优先 2 个收敛)
- **Answer**: **grid-cli + grid-server** 优先发力, grid-desktop / grid-platform / web* 静态 dormant 到下个 milestone
- **Rationale**: grid-cli 已 active (主要入口), grid-server 单租户工作台 × EAASP L1 双身份 (Leg A 服务于 EAASP 也独立运行); 完整覆盖开发者 + 单租户场景
- **Phase 4.2 影响**: ADR-V2-024 Consequences 段引用此优先级; 下个 milestone roadmap 起点 = grid-cli + grid-server engine 子任务列

### Open Item #4 — §F.Q1 vertical engine 字段 vs 厂商扩展点边界 (P1)

- **Verdict**: **yes** (从 partial 收敛)
- **Answer 1 (字段切分)**: **5+3 切分**
  - **Engine 内置 (5 项, 走 proto wire-level + Level 2/3 操作)**:
    - `priority_tier` (替代原 `dispatch_level` — 用跨行业通用名, enum: REALTIME/STANDARD/BATCH 或类似)
    - `risk_class` (high/medium/low, Level 2 — engine 读它触发 HITL gate)
    - `latency_class` (ms/s/min/h, Level 2 — engine 读它做 SLA 路由)
    - `audit_trace` (Level 3 default — engine 自动 append entries, 扩展可拦截)
    - `model_lock` (Level 3 default — engine 启动写入 hash + tool call 验证, 扩展可重写策略 eg. 信创多模型 fallback)
  - **厂商写 (3 项, 走 vendor skill / Path D)**:
    - SCADA 接入适配 (电力 vertical-specific)
    - 业务 KPI 定义 (行业 vertical-specific)
    - 行业合规表 (eg. 国调中心规范条款 — 电力 vertical-specific)
- **Answer 2 (架构承诺)**: EAASP L3/L4 必须提供 **extension hook API**, 让行业扩展 (vendor / vertical plugin) 可访问并控制 5 个 engine 内置字段。详细 API 不在 ADR-V2-024 定义, 推到 Phase 4.3+ 独立 ADR (合并 audit §3.3 Q3 ⚫ 6 项接入位 ADR 候选)
- **Rationale**: 跨行业可复用字段 → engine 内置 (cross-vertical primitive); 行业 specific 业务规则 → 厂商写 (vertical extension); 但 engine 必须支持扩展机制让 vendor 可重写 default — 这是 first-class architectural requirement
- **Phase 4.2 影响**: ADR-V2-024 Decision 段提"5+3 切分 + L3/L4 extension hook 架构承诺"; Consequences 段提 Phase 4.3+ 独立 ADR (extension hook API + 6 项接入位 contract)
